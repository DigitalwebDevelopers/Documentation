Components and Props

Taught By Code with Deepanshu Knox

React is all about components. A component is a reusable piece of UI that can be as small as a button or as big as an entire page. To make components dynamic, React uses props (short for properties), which let you pass data from one component to another.

ðŸ”¸ Creating a Component

Components can be created using simple JavaScript functions. These functions return JSX, which describes the UI. 

function Welcome() {
  return <h1>Hello, World!</h1>;
}

You can render this component like so:

<Welcome />

React components must start with a capital letter, otherwise React will treat them like regular HTML tags.

ðŸ”¸ Using Props to Pass Data

Props allow you to send information into a component. You can treat them like function arguments. 

function Greeting(props) {
  return <h2>Hello, {props.name}!</h2>;
}

<Greeting name="Alice" />

ðŸ”¸ Destructuring Props

You can make your code cleaner by using ES6 destructuring: 

function Greeting({ name }) {
  return <h2>Hello, {name}!</h2>;
}

ðŸ”¸ Composing Multiple Components

Components can include other components, making your UI easy to manage. 

function App() {
  return (
    <div>
      <Greeting name="Alice" />
      <Greeting name="Bob" />
    </div>
  );
}

ðŸ”¸ Dynamic Rendering

You can render different UI based on values, like so: This helps you build interactive, data-driven user interfaces. 

const isLoggedIn = true;
return <h1>{isLoggedIn ? 'Welcome back!' : 'Please sign in'}</h1>;

This is called composition, and it helps break down large UIs into smaller, reusable pieces.

ðŸ”¸ Conclusion of This Article

Components define the structure, and props bring in the data. By combining them, React lets you build powerful, dynamic, and reusable UIs in a clean and organized way.
