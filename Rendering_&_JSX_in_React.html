React uses a special syntax called JSX (JavaScript XML) to describe what the UI should look like. JSX lets you write HTML-like code inside JavaScript, which React then compiles into actual DOM instructions.

This makes code easier to read and write, especially when working with components.

ðŸ”¸ What is JSX ( JavaScriptXml )

JSX is a syntax extension that looks similar to HTML but is actually syntactic sugar for React.createElement() calls.

Example of JavaScript XML:

const element = <h1>Hello, world!</h1>;

This JSX gets converted to:

React.createElement('h1', null, 'Hello, world!');

You can also embed JavaScript expressions inside JSX using curly braces: 

Rconst name = 'React';
const element = <h1>Welcome, {name}</h1>;

ðŸ”¸ JSX Rules to Remember

Return only one parent element JSX must return a single enclosing tag. You can wrap elements in a <div> or a <React.Fragment>.

Use className instead of class Since JSX is closer to JavaScript than HTML, you must write className (not class). 

<div className="container">Hello</div>

Self-close empty tags without children must be self-closed: 

<input /> // self Closed Tags in react JSx

ðŸ”¸ Rendering of Elements

React elements are rendered using the ReactDOM.render() method in legacy versions, or through the createRoot() method in React 18+. 

import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);

ðŸ”¸ Dynamic Rendering

You can render different UI based on values, like so: This helps you build interactive, data-driven user interfaces. 

const isLoggedIn = true;
return <h1>{isLoggedIn ? 'Welcome back!' : 'Please sign in'}</h1>;

ðŸ”¸ JSX in Components

JSX is mostly used inside functional components to describe what the UI should look like: 

function Greeting(props) {
  return <h2>Hello, {props.name}!</h2>;
}
