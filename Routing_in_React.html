Routing in React

Taught By Code with Deepanshu Knox

Modern web applications are often made up of multiple views or pages. In React, routing helps you display different components based on the URL, without reloading the entire page. This is made possible using the React Router library.

ðŸ”¸ Installing React Router

To use routing, install the package:

npm install react-router-dom

Then import the necessary components in your app: 

import { BrowserRouter, Routes, Route } from 'react-router-dom';

ðŸ”¸ Basic Routing Setup

Here's how a basic routing setup looks:

<BrowserRouter> wraps your whole app. <Routes> contains individual <Route> definitions. path defines the URL, and element defines what to render. 

import React from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function Home() {
  return <h2>Home Page</h2>;
}

function About() {
  return <h2>About Page</h2>;
}

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

ðŸ”¸ Navigating Between Pages

To navigate between routes, use the Link component instead of regular anchor tags: 

import { Link } from 'react-router-dom';

<nav>
  <Link to="/">Home</Link>
  <Link to="/about">About</Link>
</nav>

ðŸ”¸ Conclusion of This Article

Routing in React lets you build multi-page applications using single-page architecture. By using react-router-dom, you can manage navigation, page views, and URL changes with ease and flexibility.
