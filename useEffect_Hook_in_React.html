Reactâ€™s useEffect() hook is one of the most powerful and commonly used hooks. It lets you perform side effects in your functional components, such as fetching data, directly manipulating the DOM, setting up subscriptions, and more.

What is useEffect Hook in React ?

In React, a side effect is anything that affects something outside the scope of the current function â€” like network requests, timers, or manually changing the DOM. The useEffect() hook allows you to run code after the component renders (or updates), making it ideal for these kinds of tasks. 

useEffect(() => {
  // Your side effect code here

  return () => {
    // Optional cleanup code here
  };
}, [dependencies]);

Breakdown of the Given Code :

The first argument is a function where you write the side effect code. The optional return function is for cleanup (like unsubscribing or clearing timers). The dependency array tells React when to run the effect.

If you provide an empty dependency array ([ ]), the effect runs only once after the first render â€” like componentDidMount. If you omit the dependency array, the effect runs after every render. If you provide dependencies, the effect runs only when those values change.

ðŸ”¸ Basic Example : An Counter App

Here, the effect logs the current count value every time it changes. 

import React, { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log(`Count changed to: ${count}`);
  }, [count]);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}

ðŸ”¸ Cleanup in useEffect Hook

Sometimes, effects need cleanup to avoid memory leaks or unwanted behavior (e.g., timers or subscriptions). The cleanup function runs before the component unmounts or before the next effect runs. 

useEffect(() => {
  const timer = setTimeout(() => {
    console.log('Timer done!');
  }, 1000);

  return () => clearTimeout(timer);
}, []);

Always define the dependency array properly in useEffect() to avoid infinite loops. Only include the variables that your effect depends on.
