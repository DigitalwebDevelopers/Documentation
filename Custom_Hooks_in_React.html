Understanding custom Hooks Hook

React provides powerful built-in hooks like useState() and useEffect(). But as your app grows, logic can get repeated across components. Custom Hooks let you extract and reuse that logic in a clean, reusable way.

A Custom Hook is a JavaScript function whose name starts with "use" and can call other hooks. It helps you share logic between components without repeating code. Think of it as your own personalized hook made from built-in ones.

import { useState, useEffect } from 'react';
            
function useWindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);
  
  useEffect(() => {
    const handleResize = () => setWidth(window.innerWidth);
    window.addEventListener('resize', handleResize);
    
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  
  return width;
}

You can now use this like any other hook:

function MyComponent() {
  const width = useWindowWidth();
  return <p>Window width: {width}px</p>;
}

ðŸ”¸ Another Example: useToggle

This custom hook named useToggle manages a boolean state, often used for toggling things like modals, dropdowns, or switches. It uses useState to set an initial value (defaulting to false) and returns two things:

the current state and a function named toggle that flips the state between true and false. This helps simplify toggle logic across components by reusing this small, focused hook. 

function useToggle(initialValue = false) {
  const [state, setState] = useState(initialValue);

  const toggle = () => setState(prev => !prev);

  return [state, toggle];
}

Usage of the Code :

const [isOpen, toggleOpen] = useToggle();
